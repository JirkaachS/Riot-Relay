Installer and SmartScreen

Which asset should I download?

Open the latest GitHub Release, expand Assets, and select the Windows x64 EXE/NSIS package for normal installation and automatic updates. Use MSI for managed deployment or manual updates. “Source code” archives are not installers.

Windows says the app is unrecognized

Riot Relay does not claim code signing, so SmartScreen may warn about an unsigned community build. Confirm the file came directly from github.com/JirkaachS/Riot-Relay/releases, compare published hashes when available, and inspect the source/tag. Proceed only when you trust it; delete copies from mirrors, messages, or unfamiliar domains.

The installer will not run

Confirm the file finished downloading, is the x64 Windows artifact, and is not quarantined. Check Windows Security history. In managed environments, ask the administrator to deploy the MSI; do not disable endpoint protection.

Riot Client is not detected

  1. Install or repair the official Riot Client and launch it once.
  2. In Riot Relay settings, browse to the real RiotClientServices.exe location.
  3. Keep Riot Relay and Riot Client at the same privilege level; routinely running either as administrator is not recommended.
  4. Fully quit both applications, including Riot Relay in the notification area, then reopen them.

If Riot moved the executable during an update, select its new location rather than copying it.

Switching or identity verification fails

  • Finish any Riot Client update and close running games before switching.
  • For a new account, sign in through the official Riot Client, complete 2FA there, then use Sync current.
  • Verify the displayed Riot ID belongs to the intended roster entry before saving a session.
  • If an identity mismatch appears, stop. Do not rename entries or edit files to force a match; sign out, authenticate the intended account, and sync again.
  • Expired or revoked sessions require a fresh Riot sign-in and capture.
  • Third-party rank/catalog failures can be temporary and should not be treated as identity proof.

PUUID is authoritative. Riot IDs and local labels are presentation, not safe binding keys.

Vault cannot be unlocked

Check keyboard layout, Caps Lock, and the intended Windows user account. The master password has no recovery service. Do not repeatedly alter or replace vault.dat, vault.key, or key-mode files.

If Windows-protected unlock fails, use the master password when offered. Hardware, Windows account, or credential changes can invalidate a stored key without damaging a correct password-protected vault. Restore only a known-good backup of the complete matching user-data set.

Deleting the user-data directory can permanently remove saved credentials and sessions. Back it up while Riot Relay is fully closed before any destructive recovery attempt.

Update problems

EXE does not update automatically

Check connectivity to GitHub Releases, allow the current download to finish, fully quit Riot Relay from the notification area, and relaunch. If needed, download the newest EXE from the official release and install it over the existing version.

MSI remains on an older version

MSI is intended for administrator-managed or manual updates. Download/deploy the newer MSI using the same management context. Back up user data before changing deployment method.

Version still looks wrong

Confirm you opened the installed copy rather than an older portable/downloaded file. Reboot after a pending installer operation, then verify the version shown in Riot Relay.

Safe diagnostics and support

Reproduce with synthetic labels where possible. Before sharing a screenshot or issue, remove Riot IDs, login names, PUUIDs, chat content, paths containing personal names, and any QR/2FA data. Never upload vault/session files or authentication headers.

Search existing GitHub issues, then report the app version, Windows version, package type, expected/actual behavior, and sanitized steps. Security-sensitive reports belong in the private path described on the security page.
