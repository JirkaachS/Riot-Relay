Privacy disclosure

Local-first, not network-free.

Riot Relay keeps sensitive vault material on your Windows device. Features that retrieve live Riot data, catalogs, public profiles, or updates still require network requests.

Stored locallyNetwork activityUpdatesControlsDeletionScope

Data stored locally

The application may store account labels, login usernames, encrypted passwords, identity identifiers such as PUUID, captured sessions, preferences, cached catalog/account data, and vault configuration in Electron’s per-user application-data directory. Passwords are sealed in the encrypted vault; plaintext account secrets are not intentionally written to disk.

A master-password-derived key protects each vault. If you explicitly enable passwordless unlock, a vault-bound key is protected by Windows and stored locally. Windows Hello mode additionally requires a Windows consent ceremony. These modes are disabled by default.

No recovery service: Riot Relay contributors do not receive or escrow your master password. Losing it can make the vault unrecoverable.

Network activity

Riot services

When you switch, sync, load inventory/ranks, use current-session chat, or launch a supported game, Riot Relay communicates with the local Riot Client and Riot-operated services as needed. Riot receives the information normally required for those actions under Riot’s own terms and privacy practices.

Third-party data services

Requested views can retrieve cosmetic metadata from VALORANT API or CommunityDragon and may query OP.GG’s undocumented service for League rank information, with identity checks before accepting results. Public profile links to services such as U.GG, DeepLoL, OP.GG, DPM.LOL, VTL.LOL, and Tracker.gg are opened only when selected; those sites then receive ordinary web request metadata.

Presence proxy

Deceive-style presence controls use a local proxy to alter the presence published through Riot chat. Party, invite, voice, and game traffic are intended to remain separate from that presence transformation.

Update behavior

The EXE/NSIS package is recommended for automatic application updates. An update-enabled build may contact GitHub Releases to compare versions and download a new package. GitHub can receive normal connection metadata, including IP address, timestamps, and user agent. Account credentials are not required for this check.

The MSI package is intended for managed deployment or manual replacement. Administrators control how and when MSI updates are obtained and installed.

Privacy controls

  • Hide login usernames masks sign-in names in roster and detail views.
  • Hide display names masks Riot IDs and removes them from supported filenames, JSON, and image headers.
  • Export only what you intend to share; exported JSON, CSV, and images leave the encrypted vault context.
  • Disable stored-key unlock if the device or Windows account is shared.
  • Presence controls change what friends may see; they do not make network activity anonymous.

Retention and deletion

Local data remains until removed through the application or its user-data directory. Before deleting data, fully quit Riot Relay from the Windows notification area and keep any export you intentionally need. Deleting vault or session artifacts is irreversible and can remove saved accounts or require fresh sign-in.

Riot Relay cannot delete records held independently by Riot, GitHub, or third-party services. Use each provider’s controls for those records.

Scope and changes

This disclosure describes the public version 1.3.2 behavior at a practical level; source code is authoritative for a particular build. Review release notes and source changes before updating. Report a suspected privacy or security issue through the guidance on the security page, without attaching credentials, vaults, session files, logs containing tokens, or real account identifiers.
